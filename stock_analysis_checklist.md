# Checklist วิเคราะห์หุ้นก่อนเข้าซื้อ — ฉบับนักวิเคราะห์ปัจจัยพื้นฐาน

> **หลักคิด:** การลงทุนที่ดี = ธุรกิจดี + ราคาสมเหตุสมผล + มีเหตุผลว่าทำไมตลาดตีราคาผิด (variant perception)
> งานของ checklist นี้คือพยายาม "ฆ่า" ไอเดียให้ตายก่อน ถ้าฆ่าไม่ตายค่อยซื้อ

---

## ด่านที่ 0 — Pre-screen (คัดออกก่อนเสียเวลา)

ถ้าข้อใดข้อหนึ่งไม่ผ่าน ตัดทิ้งทันที ไม่ต้องวิเคราะห์ต่อ

- [ ] **อยู่ใน circle of competence ไหม** — อธิบายว่าบริษัททำเงินยังไงใน 1–2 ประโยคได้หรือเปล่า ถ้าไม่เข้าใจธุรกิจ ห้ามซื้อ
- [ ] **สภาพคล่องพอไหม** — วอลุ่มเฉลี่ยต่อวันมากพอให้เข้า-ออกโดยไม่ทำราคาพัง (สำคัญมากกับหุ้นเล็ก)
- [ ] **ไม่ใช่บริษัทที่กำลังจะล้ม** — ไม่มีสัญญาณ going concern, ไม่ได้ใกล้ผิดนัดชำระหนี้
- [ ] **งบการเงินเชื่อถือได้** — ผู้สอบบัญชีไม่มีข้อสังเกตร้ายแรง, ไม่มีประวัติแก้งบย้อนหลังบ่อย

---

## ด่านที่ 1 — คุณภาพธุรกิจ (Business Quality)

*ธุรกิจดีสำคัญกว่าราคาถูก ธุรกิจแย่ที่ซื้อถูกมักกลายเป็น value trap*

- [ ] **โมเดลธุรกิจ** — สร้างรายได้จากอะไร ลูกค้าคือใคร จ่ายเพราะอะไร
- [ ] **Moat (คูเมือง)** — มีข้อได้เปรียบที่คู่แข่งเลียนแบบยากไหม:
  - Brand / ราคาพรีเมียมที่ลูกค้ายอมจ่าย
  - Network effect (ยิ่งคนใช้มากยิ่งมีค่า)
  - Switching cost (ย้ายไปเจ้าอื่นแพง/ยุ่งยาก)
  - Cost advantage (ต้นทุนต่ำกว่าโครงสร้าง)
  - Intangibles / สัมปทาน / ใบอนุญาต
- [ ] **คุณภาพรายได้** — recurring (สม่ำเสมอ) หรือ one-off (ครั้งเดียวจบ) — recurring มีค่ากว่ามาก
- [ ] **การกระจุกตัวของลูกค้า** — พึ่งลูกค้ารายเดียว/รายใหญ่มากไปไหม (ถ้าลูกค้าเจ้าใหญ่หายไปเจ๊งไหม)
- [ ] **โครงสร้างอุตสาหกรรม** — อุตสาหกรรมกำลังโต/ทรง/ถดถอย, การแข่งขันโหดแค่ไหน, มีคนใหม่เข้ามาง่ายไหม
- [ ] **ผู้บริหาร** — ประวัติ capital allocation ดีไหม (เอากำไรไปทำอะไร), ถือหุ้นเองเยอะไหม (insider ownership), พูดตรงกับที่ทำไหม

---

## ด่านที่ 2 — สุขภาพการเงิน (Financial Health)

### งบกำไรขาดทุน (Income Statement)
- [ ] **การเติบโตของรายได้** — CAGR 3–5 ปี, สม่ำเสมอหรือกระตุก
- [ ] **Margin trend** — gross / operating / net margin *แนวโน้ม* ขึ้นหรือลง (ดูเทรนด์สำคัญกว่าเลขเดี่ยว)
- [ ] **Operating leverage** — รายได้โตแล้วกำไรโตเร็วกว่าไหม (สัญญาณ scale ดี)

### งบดุล (Balance Sheet)
- [ ] **หนี้** — Net Debt / EBITDA (เกิน 3 เท่าเริ่มเสี่ยง), D/E
- [ ] **ความสามารถจ่ายดอกเบี้ย** — Interest Coverage (EBIT / ดอกเบี้ย) ยิ่งสูงยิ่งปลอดภัย
- [ ] **สภาพคล่องระยะสั้น** — Current Ratio / Quick Ratio
- [ ] **Goodwill บวมไหม** — ถ้า goodwill เยอะจากการซื้อกิจการแพงๆ ระวัง write-off ในอนาคต

### งบกระแสเงินสด (Cash Flow) — **โกหกยากที่สุด ดูให้ละเอียด**
- [ ] **Operating Cash Flow เทียบ Net Income** — ถ้ากำไรทางบัญชีสูงแต่เงินสดไม่เข้า = คุณภาพกำไรต่ำ (ธงแดง)
- [ ] **Free Cash Flow (FCF)** — เป็นบวกและโตไหม (นี่คือเงินจริงที่ธุรกิจสร้าง)
- [ ] **Capex intensity** — ต้องลงทุนหนักตลอดเพื่ออยู่รอดไหม (ธุรกิจกินทุน)
- [ ] **การเพิ่มทุน / dilution** — จำนวนหุ้นเพิ่มขึ้นเรื่อยๆ ไหม (ออกหุ้นใหม่ = ปันเค้กชิ้นเล็กลง) — ตรวจ share count ย้อนหลัง 5 ปี

### ผลตอบแทนต่อทุน (สำคัญมาก แต่คนมองข้าม)
- [ ] **ROE / ROIC** — บริษัทใช้ทุนสร้างผลตอบแทนได้ดีแค่ไหน
- [ ] **ROIC เทียบ WACC** — ถ้า ROIC > ต้นทุนเงินทุน = บริษัทสร้างมูลค่า ถ้าต่ำกว่า = ทำลายมูลค่า (นี่คือแก่นของ "ธุรกิจดี" ในเชิงตัวเลข)

---

## ด่านที่ 3 — Valuation (แพงหรือถูก)

*ธุรกิจดีที่ซื้อแพงเกินไป ก็เป็นการลงทุนที่แย่ได้*

- [ ] **Multiple พื้นฐาน** — P/E, Forward P/E, EV/EBITDA, P/B (สำหรับกลุ่มการเงิน), P/S
- [ ] **PEG** — P/E เทียบอัตราการเติบโต (ปรับ P/E ให้ยุติธรรมกับหุ้นโต)
- [ ] **FCF Yield** — FCF ต่อหุ้น / ราคา (เทียบกับผลตอบแทนพันธบัตรได้เลย)
- [ ] **เทียบกับตัวเอง** — ตอนนี้แพง/ถูกกว่าค่าเฉลี่ย 5–10 ปีของตัวมันเองไหม
- [ ] **เทียบกับ peer** — เทียบเพื่อนร่วมอุตสาหกรรม แพงกว่า/ถูกกว่าเพราะอะไร (มีเหตุผลรองรับไหม)
- [ ] **DCF คร่าวๆ** — ประเมินมูลค่าจากกระแสเงินสดในอนาคต (ไม่ต้องเป๊ะ แค่ได้กรอบ)
- [ ] **Reverse DCF** — ราคาปัจจุบัน "แอบคิด" ว่าบริษัทจะโตกี่ %? สมเหตุสมผลไหม (เครื่องมือกันซื้อหุ้นที่ตลาดคาดหวังเว่อร์)
- [ ] **Margin of Safety** — ราคาที่จ่าย ต่ำกว่ามูลค่าที่ประเมินมากพอไหม (เผื่อพลาด) — โดยทั่วไปอยากได้ส่วนลด 20–40%

---

## ด่านที่ 4 — สร้าง Thesis (ทำไมต้องซื้อ)

*ทุกการซื้อต้องมี thesis เขียนเป็นลายลักษณ์อักษร*

- [ ] **Variant perception** — "ฉันเชื่ออะไรที่ตลาดยังไม่เชื่อ/มองข้าม" ถ้าตอบไม่ได้ แปลว่าไม่มี edge
- [ ] **Catalyst** — อะไรจะทำให้ตลาดเห็นมูลค่าที่แท้จริง และกรอบเวลาประมาณเท่าไหร่
- [ ] **สามฉากทัศน์ + ราคาเป้าหมายคร่าวๆ:**
  - Bear case (ผิดทาง เสียกี่ %)
  - Base case (น่าจะเป็น)
  - Bull case (ถูกทาง ได้กี่ %)
- [ ] **Risk / Reward** — ถ้าถูกได้เท่าไหร่ เทียบถ้าผิดเสียเท่าไหร่ — ไม่คุ้มถ้าต่ำกว่า **2:1**
- [ ] **⭐ Invalidation point (ข้อที่คนข้ามบ่อยสุด)** — "ถ้าเห็นอะไรเกิดขึ้น แปลว่าฉันคิดผิด ต้องออก" เขียนให้ชัด *ก่อน* ซื้อ ตอนที่ยังไม่มีอารมณ์เข้ามาเกี่ยว

---

## ด่านที่ 5 — จังหวะเข้า (Entry)

*แยกให้ชัดระหว่าง "จะซื้ออะไร" (คุณภาพ+ราคา) กับ "จะซื้อเมื่อไหร่" (จังหวะ)*

- [ ] **มี margin of safety ที่ราคาปัจจุบัน** — เข้าเมื่อราคาต่ำกว่ามูลค่าพอ ไม่ใช่ไล่ราคาที่วิ่งไปแล้ว
- [ ] **ทยอยเข้า (scale-in)** — แบ่งซื้อเป็นไม้ ไม่ทุ่มหมดครั้งเดียว โดยเฉพาะถ้าไม่มั่นใจจังหวะ
- [ ] **อย่ารับมีดตก** — ราคาลงแรงไม่ใช่เหตุผลซื้อ ถ้า thesis ยังไม่ยืนยัน รอสัญญาณทรงตัวก่อน
- [ ] **ระวังจังหวะรอบ earnings** — ถ้าไม่มั่นใจผลประกอบการ การเข้าก่อนประกาศงบคือการพนัน ไม่ใช่ลงทุน
- [ ] **Position size คำนวณไว้ก่อน** — จะลงเท่าไหร่ตาม conviction และ downside (ดูด่าน 6)

---

## ด่านที่ 6 — Position Sizing & Risk

- [ ] **ขนาดตาม conviction + downside** — มั่นใจมาก + ความเสี่ยงขาลงจำกัด → ลงมากขึ้น
- [ ] **เพดานขาดทุนต่อไม้** — โดยทั่วไปไม่ให้ขาดทุนต่อ position เกิน **1–2% ของพอร์ต** เพื่อให้พลาดติดกันหลายครั้งแล้วยังรอด
- [ ] **Correlation ระหว่างตัวที่ถือ** — ถือ 5 ตัวที่ขึ้นลงพร้อมกัน = ถือตัวเดียวใส่ leverage เลือกตัวที่ปัจจัยขับเคลื่อนไม่เกี่ยวกัน
- [ ] **เพดาน exposure ต่อ sector** — ไม่กระจุกอุตสาหกรรมเดียวมากเกิน

---

## ด่านที่ 7 — จังหวะออก (Exit) — วินัยตรงนี้แยกมือโปรกับมือใหม่

### เหตุผลที่ "ควร" ขาย
- [ ] **Thesis พัง** — invalidation point ที่เขียนไว้เกิดขึ้นจริง → ออกทันที ไม่ต่อรองกับตัวเอง
- [ ] **ถึง/เกินมูลค่าที่เหมาะสม** — ราคาวิ่งเกิน fair value ไปแล้ว (margin of safety หายไป กลายเป็นความเสี่ยง)
- [ ] **ปัจจัยพื้นฐานเสื่อม** — margin หด, หนี้พุ่ง, moat ถูกกัดเซาะ, คุณภาพกำไรตก
- [ ] **มีโอกาสที่ดีกว่า (opportunity cost)** — เจอตัวที่ risk/reward ดีกว่าชัดเจน และเงินมีจำกัด
- [ ] **Position ใหญ่เกินไปจากที่วิ่งขึ้น** — trim กลับมาให้สมดุลตามระดับความเสี่ยงที่รับได้

### เหตุผลที่ "ไม่ควร" ใช้ขาย (กับดักอารมณ์)
- ❌ ราคาลงแต่ thesis ยังอยู่ครบ (นี่อาจเป็นจังหวะซื้อเพิ่มด้วยซ้ำ)
- ❌ เบื่อ / ถือมานานแล้วไม่ไปไหน
- ❌ ข่าว noise รายวันที่ไม่กระทบมูลค่าระยะยาว
- ❌ "กำไรนิดหน่อยรีบขายกันเหนียว" ทั้งที่ thesis ยังไม่จบ (ตัดกำไรสั้น ปล่อยขาดทุนยาว = สูตรพอร์ตพัง)

> **หลักคิดเรื่อง stop:** แยก **price stop** (ตัดตามราคา เหมาะกับเทรด) ออกจาก **thesis stop** (ตัดเมื่อเหตุผลลงทุนพัง เหมาะกับลงทุนพื้นฐาน) โปรเจคของคุณเน้น thesis stop เป็นหลัก

---

## ด่านที่ 8 — Red Flags (เจอแล้วเพิ่มความระวังหรือตัดทิ้ง)

- 🚩 **ลูกหนี้การค้า (receivables) โตเร็วกว่ารายได้** / DSO พุ่ง — อาจดันยอดขายด้วยเครดิตหลวมหรือแต่งตัวเลข
- 🚩 **สินค้าคงคลังบวมผิดปกติ** — ของขายไม่ออก
- 🚩 **แก้งบย้อนหลัง / เปลี่ยนผู้สอบบัญชีบ่อย**
- 🚩 **เพิ่มทุน/ออกหุ้นใหม่ซ้ำๆ** — เจือจางผู้ถือหุ้นเดิมเรื่อยๆ
- 🚩 **ผู้บริหารเทขายหุ้นหนัก** โดยเฉพาะช่วงราคาสูง
- 🚩 **ผู้บริหารพูดเว่อร์ / โปรโมตหุ้นตัวเอง** มากกว่าพูดเรื่องธุรกิจ
- 🚩 **ตัวเลขดีเกินจริง** ในอุตสาหกรรมที่ margin ปกติต่ำ — สงสัยไว้ก่อน
- 🚩 **โครงสร้างซับซ้อนเกินเหตุ** — บริษัทลูกซ้อนกันจนตามเงินไม่ทัน

## ส่วนพัฒนา Agent — ข้อมูลและโครงสร้างที่ต้องเตรียม

### Data fields ที่ agent ต้องดึงต่อหุ้น 1 ตัว
| หมวด | ตัวเลขที่ต้องมี |
|------|----------------|
| ราคา/มูลค่าตลาด | ราคา, market cap, avg daily volume |
| การเติบโต | revenue CAGR 3–5 ปี, EPS growth |
| ความสามารถทำกำไร | gross/operating/net margin (พร้อมเทรนด์), ROE, ROIC |
| งบดุล | Net Debt/EBITDA, interest coverage, current ratio |
| กระแสเงินสด | OCF, FCF, FCF margin, share count 5 ปี |
| Valuation | P/E, Forward P/E, EV/EBITDA, PEG, FCF yield, P/B |
| เชิงคุณภาพ | โมเดลธุรกิจ, moat, ผู้บริหาร/insider ownership |

### แหล่งข้อมูล
- **หุ้นไทย:** SET, SETSMART, filings จาก ก.ล.ต. (56-1), งบจากตลาดหลักทรัพย์, opportunity day
- **หุ้นต่างประเทศ:** SEC EDGAR (10-K/10-Q), IR ของบริษัท, ผู้รวบรวมข้อมูลการเงิน

### แนวคิด Scoring Rubric (ให้ agent ให้คะแนน)
แบ่งเป็นหมวด ให้น้ำหนักตามความสำคัญ เช่น:
- คุณภาพธุรกิจ / moat — 25%
- สุขภาพการเงิน — 25%
- Valuation / margin of safety — 25%
- ความชัดเจนของ thesis + catalyst — 15%
- ความเสี่ยง / red flags (หักคะแนน) — 10%

> ตั้งเกณฑ์ขั้นต่ำ (เช่น ต้องได้ >70/100) จึงเข้าสู่ shortlist และ **agent ต้องอธิบายเหตุผลของทุกคะแนน** ไม่ใช่แค่ให้ตัวเลข

### รูปแบบ Output มาตรฐานที่ agent ควรส่งกลับ
1. สรุปธุรกิจ 2–3 บรรทัด
2. คะแนนแต่ละหมวด + เหตุผล
3. Thesis (variant perception + catalyst)
4. Bear / Base / Bull + ราคาเป้าหมายคร่าวๆ
5. **Invalidation point** (อะไรจะพิสูจน์ว่าผิด)
6. Red flags ที่เจอ
7. คำแนะนำ: เข้า / ผ่าน / รอ — พร้อมช่วงราคาที่น่าเข้า

### Journal & วัดผล (หัวใจของ side project นี้)
บันทึกทุกตัวที่ซื้อ: วันที่, ราคา, thesis, invalidation point → แล้วย้อนรีวิวเป็นระยะว่า thesis ถูกไหม **และเทียบผลตอบแทนกับ "ถ้าเอาเงินก้อนนั้นไปใส่ VT ช่วงเดียวกัน"** นี่คือสิ่งเดียวที่จะตอบได้จริงว่า agent มี edge หรือแค่โชค
