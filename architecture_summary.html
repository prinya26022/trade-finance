03fundamentals ต่างกันได้ แต่บรรจบที่ Fact

แต่ละ asset เก็บ raw field ของตัวเอง (StockFundamentals / CryptoFundamentals) แต่ทุกตัวสัญญาว่าจะแปลงตัวเองเป็นชุด Fact — “ตัวเลข 1 ตัว + ป้ายชื่อ” — ก่อนถึงมือชั้นบน

# domain/interfaces.py — ชั้นกลาง
class Fundamentals(ABC):
    # สัญญากลางที่ชั้นบนคุยด้วย
    def to_facts(self) -> list[Fact]: ...

# providers/stock/fundamentals.py — ชั้นล่าง
class StockFundamentals(Fundamentals):   # IS-A Fundamentals
    revenue; free_cash_flow; roic       # raw field ของหุ้นเอง
    def to_facts(self):
        return [Fact("Revenue", ...), Fact("ROIC", ...)]
provider.get_fundamentals("AAPL")
→ StockFundamentals(revenue=383e9, roic=56, …)
│ .to_facts()
▼
ชั้นบนเห็นแค่ list[Fact]
[Fact("Revenue", 383e9, "USD", "FY24"), Fact("ROIC", 56, "%")] — ไม่รู้ว่ามาจาก asset ไหน
│ ป้อน LLM
▼
LLM ชั้นบน
สรุป “แข็ง/อ่อน ตรงไหน” + ให้คะแนน rubric — ทำครั้งเดียว ใช้ได้ทุก asset

isinstance ที่เคยกลัวหายไปสนิท เพราะชั้นบนจับแค่ Fundamentals (สัญญา) ไม่เคยจับ StockFundamentals ตรงๆ = polymorphism
